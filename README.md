# 🚀 Бэкенд Таск-менеджера (Task Manager)

Надежный, масштабируемый и модульный бэкенд для приложения по управлению задачами. Разработан как демонстрация принципов чистой архитектуры (DDD-lite), асинхронного Python и грамотной настройки современной инфраструктуры.

## 🌟 Ключевые особенности

* **RESTful API**: Быстрое, полностью типизированное и задокументированное API на базе **FastAPI** и **Pydantic V2**.
* **Интеграция Telegram-бота**: Полнофункциональный бот на **Aiogram 3** (со стейт-машиной и функционалом создания задач и просмотра списков).
* **Фоновые процессы**: Система фоновых задач и рассылки напоминаний о горящих дедлайнах под управлением **Celery** и **RabbitMQ**.
* **Чистая архитектура**: Строгое разделение ответственности с изолированными слоями `Service` и `Repository`. Обмен данными идет исключительно через DTO.
* **Современные инструменты**: Сборка Docker-образов и управление зависимостями оптимизированы с помощью сверхбыстрого **`uv`**, код отформатирован линтером **`ruff`**.
* **Тестовый интерфейс**: Легковесное frontend-приложение (HTML/JS), обернутое в reverse-proxy **Nginx**, для удобного визуального тестирования API.

## 🛠 Стек технологий

* **Язык:** Python 3.12+
* **Фреймворки:** FastAPI, Aiogram 3.x
* **База данных:** PostgreSQL 16, SQLAlchemy 2.0 (asyncpg), Alembic
* **Кэш и брокер сообщений:** Redis 7, RabbitMQ 3.13
* **Фоновые задачи:** Celery (Worker & Beat)
* **Инфраструктура:** Docker, Docker Compose, Nginx

## 🏗 Структура проекта (DDD-lite)

Кодовая база разбита на бизнес-домены для обеспечения масштабируемости и удобства поддержки:

```text
src/
├── core/       # Глобальные конфиги, подключение к БД, базовый Repository, исключения
├── users/      # Домен пользователей (Models, Schemas, Repository, Service, Router)
├── tasks/      # Домен задач (Models, Schemas, Repository, Service, Router)
├── bot/        # Логика Telegram-бота (Handlers, FSM, зависимости)
├── worker/     # Задачи Celery (Напоминания о дедлайнах)
└── frontend/   # Простой HTML/JS интерфейс для тестов API
